Déployé : plusieurs commandes par client

Après presque un mois de développement

MAJ : 2 bugs ont déjà été corrigés suite au déploiement (le lien de désinscription des anciens emails ne marchait pas, et les commandes pour un RDV avec un minimum de commande plantaient)

Je viens de déployer la nouvelle version, annoncée samedi dernier.

La liste de changements est assez longue, mais voici un résumer des changements principaux :

  • Dans l'éditeur de clients, vous pouvez autoriser un client à commander plusieurs fois par semaine.
  • Dans l'éditeur de produits, vous pouvez marquer un produit comme bio (affichera le logo bio européen à côté de l'image produit).
  • Vous pouvez faire un clic droit sur les en-têtes de tableau pour afficher/masquer certaines colonnes.
  • Beaucoup d'écrans proposent maintenant de filtrer les données (titre de la page avec une icône de loupe)
  • L'"admin client" est devenue une simple étape de la semaine.
  • Retrait de la page d'accueil (l'appli s'ouvre sur la semaine en cours) et de l'admin client.
  • Différentes options proposées (paniers, stocks, produit par RDV, suivi des paiements) sont maintenant désactivées par défaut pour les nouveaux maraîchers.
  • Un panneau dédié permet d'activer ces options (cliquer sur l’icône d'engrenages dans la barre latérale).
  • Les anciens clients (désinscrits) sont listés dans la page "Anciens clients" des paramètres.
  • L'édition des textes spéciaux (pour les nouveaux clients, SMS, désinscriptions .. ) est proposée directement dans l'onglet "invitation"
  • La connexion à gmail est proposée directement dans l'onglet "invitation"
  • Le glisser/déposer des produits dans la page produit pour les réordonner devrait mieux fonctionner.
  • Correction d'un bug concernant la disposition automatique des tickets de caisses
  • Les commandes ont maintenant un numéro de commande qui est incrémenté à chaque commande. Les clients le voient aussi. Ca devrait aider en cas de confusions, surtout s'il y a plusieurs commandes par semaines pour le même client.
  • Correction d'un bug concernant l'impression de récap de cueillette sous Chrome sur Android

Si vous découvrez des bugs ou problèmes, je suis joignable par téléphone (+33628350114 , de préférence messages WhatsApp) ou email (renan.lecaro@gmail.com).

N'hésitez pas à m'écrire si quelque chose vous semble bizarre avec cette nouvelle version. J'ai des tests automatisés pour éviter les bugs les plus gênants, mais difficile de tout tester dans les moindres détails. Je compte un peu sur vous pour me signaler les problèmes que vous voyez.

Je compte passer les prochains jours à régler les éventuels problèmes introduits par cette version, et ensuite attaquer le chantier suivant : rendre l'envoi de SMS plus fiable.
